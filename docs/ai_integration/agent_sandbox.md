## 概述
AstrBot 代理沙箱（Agent Sandbox）是一个改进的代码解释器环境。它为 Agent 提供了一个安全的隔离空间，用于执行由大语言模型（LLM）生成的代码（如 Python）。该功能增强了 Agent 处理复杂数学计算、数据处理及自动化脚本编写的能力。

## 核心契约
- **代码解释器集成**：沙箱作为 Agent 的核心工具（Tool）存在，LLM 可以通过工具调用（Tool Call）触发代码执行。
- **隔离执行**：执行环境与 AstrBot 主进程隔离，确保生成的代码不会对宿主机系统安全造成直接威胁。
- **结果反馈循环**：沙箱捕获标准输出（stdout）和错误信息（stderr），并将其作为工具执行结果返回给 LLM，支持 LLM 根据错误反馈进行代码自我修正。

## 关键行为
1. **工具触发**：当 LLM 判定需要进行逻辑运算或数据分析时，生成代码块并调用沙箱工具。
2. **环境准备**：系统动态初始化沙箱上下文。
3. **执行与回收**：代码运行完成后，结果返回至 `tool_loop_agent` 流程，沙箱资源被回收。

## 变更影响分析
- **确定性增强**：通过引入沙箱，Agent 能够通过运行代码解决 LLM 擅长的“幻觉”问题（如复杂算术），提高了任务处理的确定性。
- **提示词优化**：v4.12.1 同步优化了工具调用相关的系统提示词，以引导 LLM 更准确地构造沙箱执行请求。
- **开发者契约**：插件开发者在构建 Multi-Agent 系统时，可将沙箱视为一种标准化的 `FunctionTool` 进行分发，无需自行实现安全审计逻辑。
